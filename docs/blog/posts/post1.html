What is a UUID v1?

What is a UUID v1? The Complete Guide

Published: November 28, 2025

A UUID v1 (Version 1 Universally Unique Identifier) is a 128-bit identifier defined by RFC 4122 that combines a high-precision timestamp with a node identifier to guarantee global uniqueness — even across distributed systems without coordination.

Unlike random UUID v4, a UUID v1 is time-ordered: the most significant 60 bits contain a timestamp measured in 100-nanosecond intervals since the Gregorian calendar epoch (October 15, 1582). This means you can extract the exact creation time from any v1 UUID — down to 100 ns precision.

How UUID v1 Is Structured

Every v1 UUID follows this layout:

  • Bits 0–31: time_low
  • Bits 32–47: time_mid
  • Bits 48–59: time_hi (12 bits) + version 1 (4 bits)
  • Bits 60–73: clock sequence (14 bits) + variant (10xx)
  • Bits 74–121: node (48 bits — traditionally MAC address)

Why Use UUID v1 in 2025?

Despite privacy concerns about exposing MAC addresses, UUID v1 remains extremely valuable:

  • Naturally sortable by creation time (perfect for logs, audits, message queues)
  • Monotonically increasing within a single machine
  • Excellent for database indexing when insertion order matters
  • Deterministic debugging — you can tell exactly when an ID was made

Modern implementations (like this tool) randomize the node by default, eliminating privacy risks while preserving all benefits.
